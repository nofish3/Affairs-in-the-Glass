- 全部启用酒款的中文名、英文名和常见别名均通过参数化识别回归。

- 静态扫描确认没有联网、云开发、登录或位置/相机/相册/通讯录/用户资料权限 API 调用。

- 浏览器视觉预览在 390 × 844 设备框完成首页、偏好、推荐、识别和记录页面检查。
- 360 × 800 窄屏下五个画面均无横向溢出。
- 预览页控制台除已修复的 favicon 404 外无业务错误。

## 自动化覆盖重点

- 偏好四组必选和随机维度。
- 推荐得分、酒感保护、放宽匹配、候选去重和耗尽重置。
- 第三次换杯计数。
- 全部启用酒款中文名、英文名和别名，含全角 `Ｇ＆Ｔ`、英文空格酒名、最长别名和酒名优先。
- 逗号、顿号、分号、斜杠、空格、Tab 和换行分隔。
- 未知材料不被短中文别名错误吞并。
- 记录条件必填、输入清理、唯一 ID、排序、编辑、删除、损坏数据和写入失败。
- 500 条记录、最长识别输入和推荐计算的性能边界。
- 离线架构、范围外权限、用户数据上传和固定底栏安全区的静态保障。

## 视觉验收结论

- 首页品牌和酒杯视觉锚点在第一屏清楚可见。
- 三个入口采用整行菜单，不使用卡片矩阵。
- 推荐页首屏可见客观概述、标签和完整“为什么适合你”。
- 固定底部操作不遮挡首屏核心信息。
- 纸纹强度不影响正文可读性。
- 记录长标题和多行感受无横向溢出。
